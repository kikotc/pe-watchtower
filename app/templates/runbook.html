🔴 Alert: Service Down Critical

Triggered when GET /health returns non-200 or times out. Fires to Discord within 30 seconds.

  1. 1
    Check the dashboard health banner

    Go to /dashboard. If the banner is red, the service cannot reach itself. If it's green, the alert may have been transient.

  2. 2
    Check if the process is running 
    ps aux | grep run.py
# or if using Docker:
docker ps
  3. 3
    Check the logs for the crash reason

    Go to /dashboard → Live Log Feed, or read the file directly:

    tail -50 logs/app.log

    Look for ERROR or Exception entries just before the service went silent.

  4. 4
    Check if the database is reachable 
    docker ps | grep postgres
# if container is stopped:
docker start watchtower-db
  5. 5
    Restart the service

    See the Restart section below.

⚠️ Alert: High Error Rate Warning

Triggered when ≥50% of requests in a 2-minute window return 5xx. Fires to Discord with counts.

  1. 1
    Check the Error Rate card on the dashboard

    Go to /dashboard. The Error Rate card will be red if it's still elevated. Check the Error Rate History chart to see when it started.

  2. 2
    Find the failing requests in the log feed

    Filter the Live Log Feed for ERROR and 5xx entries. Note which paths are failing.

  3. 3
    Check for a bad deployment or recent code change 
    git log --oneline -10

    If a recent commit introduced the errors, roll back:

    git revert HEAD
python3 run.py
  4. 4
    Check database connectivity

    Most 5xx errors are database failures. Confirm the DB is up and the tables exist:

    docker ps | grep postgres

General Debugging Steps

  1. 1
    Look at structured logs

    The dashboard /logs endpoint returns the last 200 log entries. The log file at logs/app.log has full history. Every request is logged with method, path, status, and duration.

  2. 2
    Check system metrics

    Hit /metrics or check the dashboard rings. High CPU (>90%) or RAM (>90%) can cause slow responses and timeouts.

  3. 3
    Check the Top Endpoints panel

    Is one path getting hammered abnormally? That could indicate a stuck client, a loop, or a DDoS.

  4. 4
    Check recent events

    The /events endpoint shows the last 100 data-mutation events. Unexpected deletions or mass-creates can surface here.

Restarting the Service

Stop the running process (Ctrl+C or kill the PID), then:

cd /path/to/pe-watchtower
python3 run.py

If using Docker Compose:

docker compose restart app

Confirm recovery by hitting /health — it should return {"status": "ok"} within a few seconds.
